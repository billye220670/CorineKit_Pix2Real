3.对应的模式及UI说明：

基本信息：

左侧面板Tab名称：黑兽换脸

是否需要遮罩编辑器：不需要，隐藏对应按钮。

是否支持提示词反推：需要保持和其他模式一致。

是否需要输入提示词：不需要。该模式下卡片不需要显示提示词输入框。



4.UI详情：

该模式比较特殊，照片墙被分为左右两部分，左:右=3:7 我建议你可以放两个照片墙容器左右排布。 区域容器使用圆角。

左边的是用来存放换脸的脸图的，这个面板里的卡片不显示生成按钮。右侧是放要把脸换上去的照片的，这里面的卡片保持和二次元转真人模式一致，只不过不显示提示词输入框了（注意对应占位也没了）。



5.交互方式：用户把左边的脸图拖拽到右侧对应的图片卡片上，会直接出发右侧对应卡片开始生成。注意拖拽时hover到对应右侧卡片要响应动画给用户提示。开始生成的参数看到这里你也应该明白了，右侧对应卡片开始生成后和之前的流程就一样了，进入queue、xx%等等。。。

生成好后右侧对应卡片下方也和二次元转真人模式一样显示缩略图导航。



6.图片导入：用户拖拽导入和其他模式一样，只不过这里用户可以分别拖拽到左区（脸图区）和右区（目标图区）。



7.关于其他模式往这个tab拖拽：全部默认加到右区。



8..关于该模式下多选的处理：用户长按哪个面板内卡片触发的多选，就只会触发那个区域的多选。意思就是如果用户长按左区一个卡片进入多选状态，则只有左区的图片进入多选状态。右侧不参与多选。 同理如果多选状态是由右侧卡片长按后激活的话，则左侧不参与多选。

多选后，用户的操作和其他模式一致。

举例：用户长按左侧区域的脸图会激活左侧面板进入多选状态，这时用户加选三个左侧的脸图，然后拖拽任意一个到右侧区域的一个目标图上 相当于 加了三个任务到队列，分别用右边拖到的目标图和左侧的三个选中的脸图做参。 生成结束后右侧面板的那个目标图下方缩略图导航会出现3+缩略图 并自动选中最后的一个生成结果。（如果这部分逻辑已经在了则你可以忽略我只是doublecheck）
