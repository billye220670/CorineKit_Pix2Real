## 1. 项目背景

### 1.1 产品简介

CorineKit Pix2Real 是一个基于 ComfyUI 的图像生成与处理应用，面向二次元图像创作场景，提供从文生图到风格转换、精修放大等全流程能力。

### 1.2 技术架构

- **前端**：React + TypeScript + Vite
- **后端**：Express + TypeScript
- **通信**：REST API + WebSocket（实时进度推送）
- **AI 能力**：Grok API（已集成，用于提示词处理和图像反推）
- **图像引擎**：ComfyUI（本地部署）

### 1.3 现有能力

| 能力模块 | 说明 |
|---------|------|
| 工作流系统 | 10+ 图像/视频处理工作流（二次元转真人、真人精修、ZIT快出、高清重绘、视频生成等） |
| 模型选择 | 支持 Checkpoint 模型下拉选择，已维护模型元数据（昵称、分类、缩略图） |
| LoRA 组合 | 支持最多 5 个 LoRA 同时启用，每个 LoRA 可配置触发词和权重 |
| 提示词助手 | 基于 Grok API 的提示词转换和扩写功能 |
| 会话管理 | 支持会话持久化、输出文件管理 |
| 收藏系统 | 模型/LoRA 星标收藏（前端 localStorage） |

## 2. 功能目标

### 2.1 核心价值

通过引入 AI Agent，让用户以自然语言对话的方式完成图像生成和处理任务，降低操作门槛，提升使用效率。

### 2.2 核心能力

1. **轻量交互**：右下角星芒浮标唤起，结果直接体现在照片墙
2. **自然语言理解**：理解用户的图像生成/处理意图
3. **智能模型推荐**：根据需求自动选择基础模型和 LoRA 组合
4. **工作流编排**：自动规划并执行多步骤任务链
5. **图片上传与处理**：支持用户上传图片进行 Inpainting 等操作
6. **智能推荐**：基于用户历史行为提供个性化建议
7. **可扩展能力注册**：支持后续新增工作流、Tab、处理能力

### 2.3 预期收益

- 用户操作步骤减少 50% 以上
- 新用户上手时间缩短
- 复杂多步任务自动化执行

## 4. P0 - 基础建设（必须先完成）

### 4.1 智能推荐基建

#### 4.1.1 生成记录日志系统

**需求：** 每次工作流执行时，记录完整的生成配置和结果。

**数据内容：**
```typescript
interface GenerationRecord {
  id: string;
  userId: string;
  sessionId: string;
  timestamp: number;
  workflowId: number;
  workflowName: string;
  config: {
    model: string;
    loras: Array<{name: string; strength: number}>;
    prompt: string;
    negativePrompt: string;
    params: {
      width: number;
      height: number;
      steps: number;
      cfg: number;
      sampler: string;
      scheduler: string;
    };
  };
  result: {
    imageId: string;
    tabId: number;
    outputPath: string;
  };
  metadata: {
    isFavorited: boolean;      // 是否被爱心收藏
    favoriteTime?: number;     // 收藏时间
  };
}
```

**存储方式：**
- 后端：存储到文件系统或轻量级数据库（如 SQLite）
- 路径：`sessions/{sessionId}/generation-log.json`

**接口：**
- `POST /api/agent/log-generation` - 记录生成日志
- `GET /api/agent/generation-history` - 获取生成历史

#### 4.1.2 爱心收藏功能

**需求：** 在照片墙每张卡片的右下角添加爱心按钮，用户可点击收藏喜欢的图片。

**交互设计：**
- 位置：卡片右下角
- 图标：空心爱心（未收藏）/ 实心爱心（已收藏）
- 点击：切换收藏状态
- 悬停：显示"收藏"提示

**数据关联：**
- 收藏时记录关联的生成配置
- 用于后续偏好分析

**接口：**
- `POST /api/agent/favorite` - 收藏/取消收藏
- `GET /api/agent/favorites` - 获取收藏列表

#### 4.1.3 用户画像系统

**需求：** 基于生成历史和收藏数据，构建用户偏好画像。

**画像结构：**
```typescript
interface UserPreferenceProfile {
  userId: string;
  updatedAt: number;
  
  // 模型偏好（按使用频率和收藏加权）
  modelPreferences: Array<{
    model: string;
    usageCount: number;
    favoriteCount: number;
    weight: number;  // 0-1，计算得出
  }>;
  
  // LoRA 偏好（按类别分组）
  loraPreferences: {
    characters: Array<{lora: string; count: number; weight: number}>;
    poses: Array<{lora: string; count: number; weight: number}>;
    expressions: Array<{lora: string; count: number; weight: number}>;
    styles: Array<{lora: string; count: number; weight: number}>;
  };
  
  // 参数偏好
  paramPreferences: {
    preferredRatio: string;      // 如 "3:4"
    preferredSteps: number;      // 如 35
    preferredSampler: string;    // 如 "dpmpp_2m"
    preferredCFG: number;        // 如 7
  };
  
  // 风格标签偏好（从提示词提取）
  stylePreferences: Array<{
    tag: string;
    count: number;
    category: 'character' | 'pose' | 'style' | 'quality';
  }>;
  
  // 时间模式
  usagePatterns: {
    peakHours: number[];         // 活跃时段，如 [20, 21, 22]
    avgDailyGenerations: number; // 日均生成数
    lastGenerationTime: number;  // 上次生成时间
  };
  
  // 常用组合
  frequentCombinations: Array<{
    model: string;
    loras: string[];
    count: number;
  }>;
}
```

**计算逻辑：**
- 权重 = 使用次数 * 1 + 收藏次数 * 3
- 定期更新（每次生成后或定时任务）

**接口：**
- `GET /api/agent/user-profile` - 获取用户画像
- `POST /api/agent/update-profile` - 更新用户画像（后端自动调用）

#### 4.1.4 特征提取服务

**需求：** 从提示词中提取风格标签，用于偏好分析。

**提取内容：**
- 角色名（如：菲谢尔、安琪拉）
- 姿势关键词（如：壁尻、战败）
- 表情关键词（如：阿嘿颜、嫌弃脸）
- 风格标签（如：cyberpunk、写实）
- 质量标签（如：masterpiece、best quality）

**实现方式：**
- 规则匹配 + LLM 辅助提取
- 维护关键词映射表

### 4.2 Agent 基础交互

#### 4.2.1 星芒浮标（AgentFab）

**需求：** 固定在右下角的入口按钮。

**设计规范：**
- 位置：界面右下角，固定定位
- 图标：Sparkles（星芒）图标
- 尺寸：48x48px
- 样式：与主题色一致，悬停发光效果
- z-index：最高层级

**交互：**
- 点击：弹出输入对话框
- 执行中：显示加载动画（可选）

#### 4.2.2 输入对话框（AgentDialog）

**需求：** 轻量级输入弹窗。

**设计规范：**
- 位置：浮标上方，右对齐
- 尺寸：宽度 360px，高度自适应
- 背景：半透明遮罩 + 毛玻璃效果
- 关闭：点击外部或 ESC 键

**内容：**
- 输入框：多行 textarea，2-3 行高度
- 图片上传：支持粘贴、拖拽、点击选择
- 发送按钮：Send 图标
- 关闭按钮：X 图标

**图片上传：**
- 粘贴：Ctrl+V 直接粘贴剪贴板图片
- 拖拽：拖拽图片到输入框
- 点击：点击上传按钮选择文件
- 预览：显示已上传图片缩略图

#### 4.2.3 执行过程反馈

**需求：** 在对话框显示执行状态。

**状态文本：**
- "正在分析您的需求..."
- "正在生成提示词..."
- "正在调用 [ZIT快出] 工作流..."
- "正在生成图片... 45%"

**简化方案：** 如实现复杂，可简化为 "正在处理中，请稍候..."

#### 4.2.4 完成通知与跳转

**需求：** 任务完成后，自动导航到结果位置。

**完成提示：**
- 显示："已完成！生成了 1 张图片"
- 按钮："前往查看"

**跳转逻辑：**
1. 关闭对话框
2. 切换到结果所在的 Tab
3. 滚动到新生成的卡片位置（卡片居中）
4. 新卡片闪烁高亮 2-3 秒

**多结果处理：**
- 显示多个"前往查看"按钮
- 标注对应的 Tab 名称

### 4.3 意图理解与执行

#### 4.3.1 意图解析

**解析维度：**

| 维度 | 说明 | 示例 |
|------|------|------|
| 任务类型 | 生成/编辑/处理 | 生成图片、编辑图片、放大图片 |
| 角色 | 具体人物角色名 | 菲谢尔、安琪拉 |
| 姿势 | 特定姿势关键词 | 壁尻、战败 |
| 表情 | 面部表情 | 阿嘿颜、嫌弃脸 |
| 风格 | 画面风格 | 赛博朋克、写实 |
| 质量 | 质量要求 | 快速出图、高质量 |
| 后续处理 | 是否需要后续工作流 | 放大、精修 |

#### 4.3.2 模型/LoRA 推荐

**推荐逻辑：**
- 基础模型：根据风格选择（光辉/PONY）
- LoRA 组合：角色 + 姿势 + 风格（避免同类型重复）
- 自动注入触发词

#### 4.3.3 工作流执行

**执行流程：**
1. 解析意图
2. 推荐方案（如需要用户确认则显示）
3. 调用工作流 API
4. WebSocket 监听进度
5. 完成后通知用户

**复用接口：**
- `POST /api/workflow/{id}/execute`
- WebSocket `/ws`

## 5. P1 - 功能增强

### 5.1 智能推荐 V1

#### 5.1.1 增强系统提示词

**需求：** 在系统提示词中注入用户偏好，让 LLM 更智能。

**提示词模板：**
```markdown
你是 CorineKit 的 AI 图像生成助手。

## 当前用户偏好画像
- 常用模型: {{topModels}}
- 偏好角色: {{favoriteCharacters}}
- 偏好姿势: {{favoritePoses}}

3. 如果用户偏好高质量，默认推荐高清放大

## 可用工作流
- 7: ZIT快出（文生图）
- 0: 二次元转真人
- 2: 精修放大
- 10: 区域编辑
...
```

**需求：** 打开 Agent 对话框时，根据用户画像显示个性化建议。

**建议类型：**
- 基于高频使用的模型："用 {{model}} 生成..."
- 基于收藏的 LoRA："生成 {{character}} 的新图"
- 基于时间模式："继续上次的创作？"
- 基于常用组合："一键生成 {{combination}}"

**展示方式：**
- 对话框顶部显示 3-5 个快捷建议按钮
- 点击后直接填充输入框并执行

**支持方式：**
- 粘贴：Ctrl+V

**场景：**

**复用：** Tab 10 区域编辑的自动识别能力

## 6. P2 - 高级功能

### 6.1 可扩展性设计

#### 6.1.1 能力注册机制

**需求：** 新增工作流时无需修改 Agent 核心代码。

**工作流描述文件（YAML）：**
```yaml
id: 0
name: 二次元转真人
type: style_transfer
description: 将动漫风格的图片转换为写实真人风格
input:
  - type: image
    required: true
    description: 待转换的动漫图片
  - type: text
    name: prompt
    required: false
output:
  - type: image
triggers:
  keywords:
    - 二次元转真人
    - 动漫转真人
  patterns:
    - "把这张.*转成真人"
compatible_with: [2]  # 可与放大工作流串联
```

**注册表：** `workflow-registry/*.yaml`

**加载方式：** Agent 启动时自动扫描加载

### 6.2 智能推荐 V2

#### 6.2.1 实时输入补全

**需求：** 用户输入时实时推荐完整描述。

**示例：**
```
用户输入："生成一张菲"
推荐：
- "生成一张菲谢尔的图"
- "生成一张菲律宾风格的图"
```

#### 6.2.2 智能参数推荐

**需求：** 基于用户偏好自动推荐完整配置。

**推荐内容：**
- 模型
- LoRA 组合及强度
- 生成参数（比例、步数、CFG）
- 优化后的提示词

#### 6.2.3 组合推荐

**需求：** 发现用户的常用组合并主动推荐。

**示例：**
```
"检测到您经常使用 菲谢尔+壁尻+高质量 的组合，一键生成？"
```

### 7.1 复用现有接口

| 接口 | 用途 |
|------|------|
| `POST /api/workflow/{id}/execute` | 执行工作流 |

| `GET /api/models/metadata` | 获取模型元数据 |
| WebSocket `/ws` | 实时进度推送 |

### 7.2 新增接口

| 接口 | 用途 | 优先级 |
|------|------|--------|
| `POST /api/agent/log-generation` | 记录生成日志 | P0 |
| `GET /api/agent/generation-history` | 获取生成历史 | P0 |
| `POST /api/agent/favorite` | 收藏/取消收藏 | P0 |
| `GET /api/agent/favorites` | 获取收藏列表 | P0 |
| `GET /api/agent/user-profile` | 获取用户画像 | P0 |
| `POST /api/agent/chat` | Agent 对话入口 | P0 |
| `POST /api/agent/execute` | 执行 Agent 任务 | P0 |

### 7.3 LLM 配置

**推荐模型：** Grok 4-1 Non-reasoning

**选择原因：**
- 意图识别任务相对简单
- 响应速度快
- Token 消耗低
- 支持 Function Calling 和 Structured Outputs

**可选功能：**
- Function Calling：让 LLM 直接返回函数调用
- Structured Outputs：确保响应格式一致

## 10. 非功能需求

| 指标 | 要求 |
|------|------|
| 意图解析响应时间 | < 2 秒 |
| 生成记录存储 | 不阻塞主流程，异步写入 |
