## 简介

CorineKit Pix2Real 是一个基于 ComfyUI 的 AI 图像生成系统，提供了完整的 RESTful API 路由体系。该系统通过精心设计的路由架构，实现了从工作流执行、会话管理到 AI Agent 对话的全方位功能支持。

系统采用模块化设计，每个路由模块负责特定的功能领域，通过统一的错误处理机制和标准化的响应格式，确保了系统的稳定性和可维护性。

## 性能考虑

### WebSocket 进度优化

系统通过 WebSocket 实现实时进度监控，采用了多种优化策略：

1. **事件缓冲机制**：在客户端连接之前捕获的进度事件会被缓冲，确保客户端连接后能收到完整的进度信息
2. **权重化进度计算**：基于节点类型和执行步骤计算权重，提供更准确的全局进度
3. **多轮场景支持**：支持 tiled sampler 等多轮执行场景的进度追踪

### 文件上传优化

工作流路由实现了高效的文件上传处理：

1. **内存存储**：使用 multer 的内存存储策略，减少磁盘 I/O
2. **文件类型验证**：严格的文件类型验证，防止恶意文件上传
3. **尺寸限制**：合理的文件大小限制，平衡质量与性能

### 缓存策略

Agent 路由实现了元数据缓存机制：

- **缓存 TTL**：元数据缓存 1 分钟，平衡新鲜度与性能
- **按需加载**：只有在需要时才读取和解析元数据文件

## 故障排除指南

### 常见问题诊断

#### ComfyUI 连接问题

**症状**：工作流执行失败，返回 "Queue prompt failed" 错误

**解决方案**：
1. 检查 ComfyUI 服务是否正常运行
2. 验证 ComfyUI URL 配置是否正确
3. 确认防火墙设置允许本地连接

#### 文件上传失败

**症状**：上传文件时返回 500 错误

**解决方案**：
1. 检查磁盘空间是否充足
2. 验证文件权限设置
3. 确认文件大小未超过限制

#### WebSocket 连接中断

**症状**：进度监控中断，无法获取实时进度

**解决方案**：
1. 检查网络连接稳定性
2. 验证 WebSocket 端口是否被防火墙阻止
3. 确认客户端与服务器的时间同步

### 日志分析

系统提供了详细的日志记录，有助于问题诊断：

- **服务器启动日志**：显示服务器配置和启动状态
- **路由处理日志**：记录每个请求的处理过程
- **WebSocket 事件日志**：跟踪进度事件的传递
- **错误日志**：记录异常情况和错误详情
